📡 Local API Document

Local HTTP interface for task management & data retrieval

🔗 http://127.0.0.1:52738/api
📦 Universal Response Format

All endpoints return a JSON structure defined by ResultInfo class. The actual payload is inside Data field as a JSON string.

所有接口均返回 ResultInfo 类定义的 JSON 结构。业务数据位于 Data 字段内（JSON 字符串）。

{
  "Success": boolean,
  "Data": string,       // JSON serialized result
  "Error": string | null
}
✔️ Always parse Data with JSON.parse() to access inner objects/arrays.
✔️ 请使用 JSON.parse() 解析 Data 字段以获取实际对象或数组。
📋 1. Get all tasks
📋 1. 获取所有任务列表
/task/list
POST

Retrieve every task registered in the system. Response Data contains JSON array of TaskInfo objects.

获取系统中所有已注册任务。Data 字段返回 TaskInfo 对象的 JSON 数组字符串。

📥 Request (empty object)

{}

📤 Response Example (Success)

{
  "Success": true,
  "Data": "[{\"TaskID\":\"T1775716725005\",\"TaskName\":\"fb_searchbykeywords_20260430\",\"TaskDescription\":\"Scrape FB keywords\",\"TaskLastRunTime\":\"2026-04-30T10:00:00\",\"TaskType\":1,\"Platform\":2,\"CollectType\":0}]",
  "Error": null
}

🧪 cURL Example

curl -X POST http://127.0.0.1:52738/api/task/list \
  -H "Content-Type: application/json" \
  -d '{}'
📘 TaskInfo fields: TaskID, TaskName, TaskDescription, TaskLastRunTime (ISO datetime), TaskType (int), Platform (int), CollectType (int).
📘 TaskInfo 字段：TaskID， TaskName， TaskDescription， TaskLastRunTime（ISO 时间），TaskType（整型），Platform（整型），CollectType（整型）。
📎 2. Copy task
📎 2. 复制任务
/task/list
POST

Create a new task by duplicating an existing one. Returns the new task ID.

通过复制已有任务来创建新任务。返回新任务 ID。

📥 Request Parameters

FieldTypeRequiredDescription
taskIdstringYesOriginal task ID原始任务ID
taskNamestringYesNew task name新任务名称
parameterstringYesCustom parameter (e.g., "ai")自定义参数（例如 "ai"）
{
  "taskId": "T1775716725005",
  "taskName": "fb_searchbykeywords_20260430",
  "parameter": "ai"
}

📤 Response Example (Success)

{
  "Success": true,
  "Data": "T1775716725123",
  "Error": null
}

🧪 cURL Example

curl -X POST http://127.0.0.1:52738/api/task/list \
  -H "Content-Type: application/json" \
  -d '{"taskId":"T1775716725005","taskName":"cloned_task_01","parameter":"ai"}'
⚙️ 3. Operate task (status / run / stop)
⚙️ 3. 操作任务（获取状态/运行/停止）
/task/list
POST

Perform operations using integer operate based on TaskOpeareteType.

使用整数 operate（基于 TaskOpeareteType 枚举）执行操作。

ParameterTypeDescription
taskIdstringTarget task ID目标任务ID
operateint0 = GetStatus, 1 = RunTask, 2 = StopTask0 = 获取状态, 1 = 运行任务, 2 = 停止任务
{
  "taskId": "T1777536544335",
  "operate": 0
}

📤 Responses

📤 响应说明

  • operate = 0 (GetStatus) → Data returns task state: NotRun, Completed, Waiting, Running, Stopped.
  • operate = 0 (获取状态) → Data 返回任务状态字符串：NotRun, Completed, Waiting, Running, Stopped。
  • operate = 1 or 2 (RunTask / StopTask) → Data returns "1" on success.
  • operate = 1 或 2 (运行/停止) → 成功时 Data 返回 "1"。

✅ GetStatus response example

{
  "Success": true,
  "Data": "Running",
  "Error": null
}

▶️ RunTask response example

{
  "Success": true,
  "Data": "1",
  "Error": null
}

🧪 cURL Example (GetStatus)

curl -X POST http://127.0.0.1:52738/api/task/list \
  -H "Content-Type: application/json" \
  -d '{"taskId":"T1777536544335","operate":0}'
💡 Change operate to 1 (run) or 2 (stop) for start/stop actions.
💡 将 operate 改为 1（运行）或 2（停止）可执行对应操作。
📄 4. Fetch task results
📄 4. 获取任务执行结果
/task/getresult
POST

Retrieve incremental results of a completed/running task. Supports filtering by startTime (ISO 8601) or startId (record id).

获取已完成或运行中任务的增量结果。支持按 startTime（ISO 8601）或 startId（记录ID）筛选。

FieldTypeRequiredDescription说明
taskIdstringYesTask identifier任务ID
startTimestring (ISO) or nullNoFetch results after this timestamp获取此时间之后的结果
startIdintNoFetch results after this record ID获取此ID之后的结果
{
  "taskId": "T1777536544335",
  "startTime": null,
  "startId": 0
}

📤 Response Example

{
  "Success": true,
  "Data": "[{\"Id\":101,\"Content\":\"sample post text\",\"Author\":\"user123\"},{\"Id\":102,\"Content\":\"another result\"}]",
  "Error": null
}

Note: The actual type inside the array depends on the task type: ContentInfo[], PostInfo[] or UserInfo[]. Use the provided fields accordingly.

注意： 数组内实际类型取决于任务类型：可能是 ContentInfo[]、PostInfo[] 或 UserInfo[]，请按实际情况处理。

🧪 cURL Example

curl -X POST http://127.0.0.1:52738/api/task/getresult \
  -H "Content-Type: application/json" \
  -d '{"taskId":"T1777536544335","startTime":null,"startId":0}'
📌 To fetch after a specific time: "startTime": "2026-05-01T00:00:00"
📌 如需获取指定时间之后的结果，传递 "startTime": "2026-05-01T00:00:00"
